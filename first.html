HTML

HTML은 하이퍼텍스트 마크업 언어 (HyperText Markup Language, 문화어: 초본문표식달기언어, 하이퍼본문표식달기언어)라는 의미의 웹 페이지를 위한 지배적인 마크업 언어다. HTML은 제목, 단락, 목록 등과 같은 본문을 위한 구조적 의미를 나타내는 것뿐만 아니라 링크, 인용과 그 밖의 항목으로 구조적 문서를 만들 수 있는 방법을 제공한다. 그리고 이미지와 객체를 내장하고 대화형 양식을 생성하는 데 사용될 수 있다. HTML은 웹 페이지 콘텐츠 안의 꺾쇠 괄호에 둘러싸인 "태그"로 되어있는 HTML 요소 형태로 작성한다.

HTML은 웹 브라우저와 같은 HTML 처리 장치의 행동에 영향을 주는 자바스크립트와 본문과 그 밖의 항목의 외관과 배치를 정의하는 CSS 같은 스크립트를 포함하거나 불러올 수 있다. HTML과 CSS 표준의 공동 책임자인 W3C는 명확하고 표상적인 마크업을 위하여 CSS의 사용을 권장한다.

태그를 사용하는 것의 중요성

1. 웹페이지에 담긴 HTML 코드를 분석 -> 태그에 근거해 정리 정돈 한다.
h(제목태그)로 표시된 제목을 기준으로 검색엔진에 검색된다. = 제목처럼 보이는 제목(강조와 px사이즈를 지정)과 태그로 지정된 제목을 비교하면 검색엔진에서 후자가 먼저 노출된다.
내용에 대해서도 태그의 의미에 맞게 작업을 해야 웹페이지가 더 가치 있게 된다.

2.Accessibility(개방성, 접근성)

웹페이지는 오픈소스 이므로 신체적 장애가 있는 사람들도 정보로부터 소외당하지 않도록 해야한다. 따라서 문자를 이미지로 만들게 된다면 시각장애인분들에게는 없는 정보가 된다.

이처럼 HTML은 비즈니스 측면이 아니라 휴머니즘 측면에서도 매우 중요한 기술이다.

부모 자식과 목록

태그가 서로 포함관계에 있을때 포함하고 있는 태그를 부모태그, 포함된 태그를 자식태그라 한다. 하지만, 부모 자식관계가 바뀌기도 한다.
자식이 있는 곳에 항상 부모가 있고, 부모가 있는 곳에 항상 어떤 특정한 자식 태그가 있는 태그들이 있다.

문서 구조


본문을 설명하는 부분을 묶어 head부분이라고 부르고 본문을 body라고 정해서 태그를 이용해 묶어준다.
모든 html에는 head와 body태그가 쓰이는데 그 상위의 태그 즉, 최상위 태그는 html이다. 관용적으로 html태그 위에 !DOCTYPE HTML를 써준다.

Tip

  1. Tab키를 누르면 들여쓰기 가 된다.
  2. 목록 태그를 쓸때 또는 연속되게 태그를 쓸 경우 다중 커서를 이용해 한번에 태그를 입력 할 수 있다.
  3. Ctrl키를 누른 채로 클릭하는 위치에 다중 커서가 되어 한번에 태그를 입력 할 수 있다.
  4. 태그를 쓰고 스페이스바를 누르면 관련 속성을 추천해 준다.
